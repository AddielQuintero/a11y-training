Checkpoint 3.3.2 in the context of web accessibility refers to the Web Content Accessibility Guidelines (WCAG) Success Criterion (SC) 3.3.2, which is titled "Labels or Instructions." This criterion is part of WCAG 2.1, which is a set of guidelines published by the World Wide Web Consortium (W3C) to make web content more accessible to people with disabilities.

SC 3.3.2 requires that labels or instructions are provided when content requires user input. This means that any time a user is expected to enter information into a form or other input field, there should be clear labeling or instructions to help them understand what is required. Good Practices:

  • Use clear and concise labels for form inputs.
  • Group related form elements using <fieldset> and <legend> elements when appropriate.
  • Provide instructions or hints to help users understand how to complete the form fields.
  • Use aria-describedby to link additional descriptive information to form inputs.
  • Ensure that error messages are announced by assistive technologies and provide clear instructions on how to fix the error.

diferencia entre los checkpoints 1.4.11 y 1.4.1

BOT

Los checkpoints 1.4.11 y 1.4.1 se refieren a dos criterios de éxito diferentes dentro de las Pautas de Accesibilidad para el Contenido Web (WCAG) 2.1, que se centran en aspectos distintos de la accesibilidad visual.

Criterio de Éxito 1.4.1 Uso del Color (Checkpoint 1.4.1): Este criterio de éxito se centra en el uso del color como único medio visual para transmitir información, indicar una acción, solicitar una respuesta o distinguir un elemento visual. Según este criterio, el color no debe ser la única señal visual utilizada, ya que los usuarios que no pueden percibir el color (por ejemplo, personas con daltonismo) no podrían acceder a la información. Por lo tanto, debe haber otros indicadores visuales o textuales que proporcionen la misma información que el color.

Ejemplo de buena práctica para cumplir con el Criterio de Éxito 1.4.1:

  • Asegurarse de que los enlaces dentro de los textos se subrayen o se diferencien de alguna otra manera además del color.
  • Proporcionar etiquetas de texto o iconos adicionales para indicar errores en formularios, no solo cambiar el color de los campos de entrada.

Criterio de Éxito 1.4.11 Contraste de No Texto (Checkpoint 1.4.11): Este criterio de éxito se centra en el contraste mínimo entre los elementos gráficos y sus respectivos fondos. Esto incluye componentes de la interfaz de usuario (como botones y formularios) y elementos gráficos que son necesarios para comprender el contenido o la funcionalidad. El objetivo es asegurar que estos elementos sean visibles y distinguibles para los usuarios con problemas de visión o en condiciones de luz desfavorables. El contraste mínimo recomendado es de 3:1 para los componentes de la interfaz de usuario y los estados visuales de los elementos gráficos.

Ejemplo de buena práctica para cumplir con el Criterio de Éxito 1.4.11:

  • Asegurarse de que los botones y controles interactivos tengan un contraste suficiente con el fondo, incluso cuando se encuentren en estados como "hover" o "focus".
  • Verificar que los iconos y otros elementos gráficos esenciales tengan suficiente contraste con sus fondos para que sean fácilmente identificables.

Ambos criterios son fundamentales para garantizar que el contenido sea accesible para los usuarios con discapacidades visuales y para aquellos que operan en entornos con condiciones de iluminación subóptimas.
